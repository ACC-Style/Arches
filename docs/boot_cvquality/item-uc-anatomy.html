8.1 Anatomy

Anatomy of a Utility Class Name

The breakdown of the names for UCs follow a structured patter for easy muscle memory learning.

Example

All of the Class Names follow this ordered pattern of construction.

  1. State - Hover (Optional)
  2. Class Name
  3. Class Name Modifier (Optional)
  4. Value
  5. Value Modifier (Optional)
  6. Breakpoint (Optional)
  7. *Dividers can be : _ or - depending on usage
Example 1
Make the background color primary modified by the shading of negative 2 when the cursor has hovered over it.
I have the class h:bg_primary-n2

  1. 'h' => Interaction State
  2. `:`=> Interaction State Divider when use state interaction
  3. 'bg' => Class Name
  4. `_` => Value Divider think of this as an equal mark
  5. `primary` => Value
  6. `-` => Modifier Divider think of this as shift from the value
  7. `n2` => Modifier Values most follow a [n5 , n4, n3, n2, n1, {0 is default} , 1, 2, 3, 4, 5, color names, or shade values]
  8. `:`=> Breakpoint Divider when rule should apply
  9. `lg`=> Breakpoint Value
